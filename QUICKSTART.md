# 🚀 快速开始指南

> **5 分钟快速上手，开始优化你的任务流！**

## 📦 第一步：安装依赖

### 1. 安装 Python 依赖

```bash
pip install -r requirements.txt
```

**国内用户加速：**
```bash
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

### 2. 安装 Graphviz（必需）

**Windows:**
```powershell
# 使用 Chocolatey
choco install graphviz

# 或手动安装后添加到 PATH
```

**macOS:**
```bash
brew install graphviz
```

**Linux:**
```bash
sudo apt-get install graphviz graphviz-dev
```

**验证安装:**
```bash
dot -V
```

## ▶️ 第二步：启动应用

```bash
streamlit run app.py
```

**浏览器将自动打开到:** `http://localhost:8501`

**如果未自动打开，请手动访问上述地址。**

## 📥 第三步：导入示例数据

### 方式 A: 加载内置示例（推荐新手）

1. 点击侧边栏的 **"📝 加载示例数据"** 按钮
2. 系统自动填充预设的任务列表
3. 直接进行分析

### 方式 B: 上传 CSV 文件

1. 点击侧边栏的 **"📁 上传 CSV 文件"**
2. 选择项目中的 `example_tasks.csv` 文件
3. 查看自动解析的任务列表

### 方式 C: 手动输入任务

在侧边栏表格中直接输入：
- **任务 ID**: A, B, C...
- **耗时**: 数字
- **单位**: 从下拉框选择 (s/min/h/d等)
- **前置依赖**: 逗号分隔（如 "A, B"）

## 🔍 第四步：分析结果

点击主界面的 **"🔍 开始分析与优化"** 按钮

### 查看关键指标

| 指标 | 说明 |
|------|------|
| 📉 理论最小总耗时 | 项目最短完成时间 |
| 🔴 关键路径任务数 | 决定总工期的关键任务数量 |
| 📊 总任务数 | 所有任务的总数 |

### 查看可视化图表

#### 1. 甘特图（红色=关键路径）
- 🔴 **红色条**: 关键路径任务（不能延迟）
- 🔵 **蓝色条**: 非关键路径任务（有浮动时间）
- **X 轴**: 时间（秒）或日期（如果设置了开始日期）
- **Y 轴**: 任务列表

#### 2. 拓扑图
- **节点**: 每个任务
- **箭头**: 依赖关系
- **红色节点**: 关键路径任务
- **蓝色节点**: 非关键路径任务

### 高级功能：有限线程调度

1. 在侧边栏勾选 **"启用有限线程模式"**
2. 选择：
   - **手动指定**: 设置线程数量（如 2、4 个）
   - **自动优化**: 系统寻找最优线程数
3. 查看调度结果和线程利用率

## 💡 核心概念速查

### 🔴 关键路径（Critical Path）

**定义**: 决定项目最短工期的任务链

**特征**:
- ✅ 路径总耗时 = 项目最短工期
- ✅ 任何延迟都会影响总工期
- ❌ 没有浮动时间
- 🔴 图中用红色标识

**优化建议**:
- 优先保障资源
- 缩短任务耗时
- 拆分可并行子任务

### 🔵 非关键路径（Non-Critical Path）

**定义**: 不直接决定总工期的任务链

**特征**:
- ✅ 有浮动时间
- ✅ 可以适当延迟
- ✅ 资源紧张时可调整
- 🔵 图中用蓝色标识

**优化建议**:
- 利用浮动时间平衡资源
- 必要时延后执行
- 作为缓冲空间

## 📋 CSV 格式速查

### 标准格式

```csv
id,duration,unit,deps
A,2,h,
B,30,min,A
C,1,d,A
D,500,ms,"B, C"
```

### 字段说明

| 字段 | 必填 | 示例 | 说明 |
|------|------|------|------|
| **id** | ✅ | A, Task1 | 任务唯一标识 |
| **duration** | ✅ | 5, 3.5 | 耗时数值 |
| **unit** | ✅ | s, min, h, d | 时间单位 |
| **deps** | ❌ | A, "B, C" | 前置依赖（逗号分隔） |

### 时间单位代码

| 代码 | 含义 | 换算 |
|------|------|------|
| ms | 毫秒 | 0.001 秒 |
| s | 秒 | 1 秒 |
| min | 分钟 | 60 秒 |
| h | 小时 | 3600 秒 |
| d | 天 | 86400 秒 |
| week | 周 | 604800 秒 |
| month | 月 | 2592000 秒 |
| year | 年 | 31536000 秒 |

## 🔧 常见问题

### Q1: 提示"循环依赖"错误

**原因**: 存在环形依赖（如 A→B→C→A）

**解决**: 
1. 检查依赖关系图
2. 移除造成环路的依赖
3. 确保是有向无环图 (DAG)

### Q2: 甘特图不显示

**原因**: 没有有效任务或所有任务耗时为 0

**解决**:
1. 添加至少一个有效任务
2. 确保任务耗时 > 0
3. 刷新页面

### Q3: 拓扑图报错

**原因**: Graphviz 未安装或未配置

**解决**:
1. 安装 Graphviz
2. 添加到系统 PATH
3. 重启终端和应用

### Q4: 中文显示乱码

**原因**: CSV 文件编码问题

**解决**:
1. 使用 UTF-8 编码保存 CSV
2. Excel 另存为时选择 UTF-8

### Q5: 如何导出结果？

**方法**:
1. 截图保存图表
2. 在详细数据表中复制数据
3. 导出 CSV 备份原始数据

## 🎓 进阶技巧

### 技巧 1: 快速识别瓶颈
- 只关注**红色**的关键路径任务
- 这些任务的优化直接影响总工期

### 技巧 2: 利用浮动时间
- 蓝色任务有缓冲空间
- 资源紧张时可延后

### 技巧 3: 迭代优化
- 调整后立即重新分析
- 对比不同方案的效果

### 技巧 4: 任务拆分
- 将大任务拆分为可并行的小任务
- 增加并行度缩短总工期

### 技巧 5: 依赖优化
- 减少不必要的依赖
- 串行改并行
