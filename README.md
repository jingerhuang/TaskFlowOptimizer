## ✨ 功能特性

### 核心功能

- **🎯 关键路径自动识别**
  - 智能识别决定项目最短工期的关键任务链
  - 红色高亮显示瓶颈任务，一目了然

- **📊 交互式甘特图**
  - Plotly 驱动的动态可视化
  - 支持日期格式和时间格式双模式
  - 悬停显示详细任务信息

- **🕸️ 拓扑依赖图**
  - Graphviz 布局引擎
  - 清晰呈现任务间的依赖关系
  - 关键路径高亮显示

- **🧵 有限线程调度**
  - 支持手动指定或自动优化线程数
  - 贪心算法 + 优先级调度的启发式策略
  - 线程利用率分析与可视化

- **📁 CSV 批量导入**
  - 支持 UTF-8 编码 CSV 文件
  - 自动解析和验证数据格式
  - 快速加载大量任务数据

- **🔍 循环依赖检测**
  - 自动检测依赖环
  - 友好的错误提示和定位
  - 避免逻辑错误

### 高级特性

- **⏱️ 多时间单位支持**

- **📈 浮动时间计算**
  - 识别非关键路径任务的灵活调度空间

```bash
brew install graphviz
```

```bash
# Ubuntu/Debian
sudo apt-get update

# CentOS/RHEL
sudo yum install graphviz

sudo pacman -S graphviz
```

## 📖 使用方法

### 方式一：手动输入任务（适合少量任务）

1. **在侧边栏表格中输入任务信息**
   
   每行包含四个字段：
   - **任务 ID**: 唯一标识符（如 A, B, C 或 Task1, Task2）
   - **耗时**: 任务执行时间（数值）
   - **单位**: 时间单位（下拉选择：ms/s/min/h/d/week/month/year）
   - **前置依赖**: 前置任务 ID，多个用逗号分隔（可选）

2. **点击"🔍 开始分析与优化"按钮**
   
   系统自动计算并显示：
   - 关键路径和总工期
   - 甘特图和拓扑图
   - 详细计算数据

### 方式二：导入 CSV 文件（适合批量任务）

1. **准备 CSV 文件**
   
   使用 Excel 或文本编辑器创建 CSV 文件，格式如下：
   ```csv
   id,duration,unit,deps
   A,2,h,
   B,30,min,A
   C,1,d,A
   D,500,ms,"B, C"
   E,1.5,h,D
   ```

2. **上传文件**
   
   - 在侧边栏点击"上传 CSV 文件"
   - 选择准备好的 CSV 文件
   - 系统自动解析并显示任务列表

3. **分析和优化**
   
   - 点击分析按钮查看结果
   - 可在表格中进一步调整

### 方式三：加载示例数据（快速体验）

1. **点击"📝 加载示例数据"按钮**
   
   系统预置了一套完整的示例任务

2. **查看分析结果**
   
   直接点击分析按钮查看完整演示

### 高级功能：有限线程调度

1. **启用有限线程模式**
   
   在侧边栏勾选"启用有限线程模式"

2. **选择线程模式**
   
   - **手动指定**: 固定线程数量（如 2 个、4 个线程）
   - **自动优化**: 系统自动搜索最优线程数

3. **查看调度结果**
   
   - 每个线程的任务分配
   - 线程利用率统计
   - 总完成时间对比图

### 实用技巧

#### 技巧 1: 快速识别瓶颈
- 关注**红色标识**的关键路径任务
- 这些任务的延迟会直接影响总工期
- 优先优化这些任务的耗时

#### 技巧 2: 利用浮动时间
- 蓝色非关键路径任务有浮动时间
- 资源紧张时可延后这些任务
- 用于平衡资源分配

#### 技巧 3: 迭代优化
- 调整任务耗时后立即重新分析
- 对比不同方案的总工期
- 找到最优的资源配置

#### 技巧 4: 依赖关系优化
- 减少不必要的依赖
- 将串行改为并行
- 增加任务的并行度

## 📄 CSV 文件格式

### 标准格式

CSV 文件应包含以下四列：

```csv
id,duration,unit,deps
A,2,h,
B,30,min,A
C,1,d,A
D,500,ms,"B, C"
E,1.5,h,D
F,45,min,C
G,0.5,d,"E, F"
```

### 列说明

| 列名 | 类型 | 必填 | 说明 | 示例 |
|------|------|------|------|------|
| **id** | 字符串 | ✅ | 任务唯一标识符 | `A`, `Task1`, `设计阶段` |
| **duration** | 数值 | ✅ | 任务耗时 | `5`, `3.5`, `100` |
| **unit** | 字符串 | ✅ | 时间单位 | `s`, `min`, `h`, `d` |
| **deps** | 字符串 | ❌ | 前置依赖 ID | `A`, `"B, C"` |

### 时间单位对照表

| 单位代码 | 全称 | 中文 | 换算为秒 |
|---------|------|------|---------|
| ms | millisecond | 毫秒 | 0.001 |
| s | second | 秒 | 1 |
| min | minute | 分钟 | 60 |
| h | hour | 小时 | 3600 |
| d | day | 天 | 86400 |
| week | week | 周 | 604800 |
| month | month | 月 | 2592000 (30 天) |
| year | year | 年 | 31536000 (365 天) |

### 格式规范

- **分隔符**: 英文逗号 `,`
- **文本限定符**: 英文双引号 `"`（当字段包含逗号时）
- **编码**: UTF-8（推荐）或 GBK
- **换行**: LF (`\n`) 或 CRLF (`\r\n`)
- **空值处理**: 空单元格或留空
- **注释**: 不支持注释，所有行都会解析

## ⚠️ 常见问题

### 错误诊断与解决

#### 1. Graphviz 未安装或未配置

**错误信息**:
```
ExecutableNotFound: failed to execute ['dot', '-Kdot', ...]
Make sure the Graphviz executables are on your system's path
```

**原因**: 
- 未安装 Graphviz
- Graphviz 未添加到系统 PATH 环境变量

**解决方案**:

**Windows**:
1. 确认已安装 Graphviz
2. 找到 Graphviz 安装目录（通常 `C:\Program Files\Graphviz\bin`）
3. 将该目录添加到系统 PATH 环境变量
4. 重启终端和浏览器

**macOS/Linux**:
```bash
# 验证安装
which dot
# 应输出：/usr/bin/dot 或 /usr/local/bin/dot

# 如果找不到，重新安装
brew install graphviz  # macOS
sudo apt-get install graphviz  # Ubuntu
```

#### 2. 循环依赖检测

**错误信息**:
```
❌ 错误：检测到循环依赖！请检查任务逻辑。
```

**原因**: 
- 任务间存在环形依赖
- 例如：A 依赖 B，B 依赖 C，C 又依赖 A

**解决方案**:
1. 仔细检查每个任务的依赖关系
2. 画出依赖关系图，查找环路
3. 移除或修改造成循环的依赖
4. 确保依赖关系是有向无环图 (DAG)

**示例**:
```
错误示例：A → B → C → A
修正方案：A → B → C （C 不再依赖 A）
```

#### 3. 无效任务 ID 或依赖不存在

**错误信息**:
```
💡 请检查任务 ID 是否唯一，以及依赖的 ID 是否存在于列表中。
```

**原因**:
- 引用了不存在的任务 ID
- 任务 ID 重复
- 使用了空 ID 或 'nan'

**解决方案**:
1. 确保所有任务 ID 唯一
2. 确保依赖的 ID 在任务列表中存在
3. 避免使用空 ID 或'nan'字符串
4. 检查是否有拼写错误

#### 4. CSV 文件读取失败

**错误信息**:
```
❌ 读取 CSV 文件失败：xxx
```

**可能原因**:
- 文件编码不是 UTF-8
- CSV 格式不正确
- 缺少必要的列

**解决方案**:
1. 使用 UTF-8 编码保存 CSV 文件
2. 确认包含 `id`, `duration`, `unit`, `deps` 四列
3. 检查 CSV 格式是否符合规范
4. 避免使用特殊字符和中文标点

#### 5. 甘特图不显示

**现象**: 点击分析后甘特图为空白

**原因**:
- 没有有效任务数据
- 所有任务耗时为 0

**解决方案**:
1. 确保至少添加了一个有效任务
2. 任务耗时必须大于 0
3. 刷新页面重新加载

#### 6. 中文显示乱码

**现象**: 图表中中文显示为方框或乱码

**解决方案**:

**Windows**: 无需额外配置，系统自带中文字体

**Linux**:
```bash
# 安装文泉驿字体
sudo apt-get install fonts-wqy-zenhei

# 或安装思源字体
sudo apt-get install fonts-noto-cjk
```

**macOS**: 通常无需配置，系统自带华文黑体

### 使用技巧

#### 提高计算性能
- 任务数量较多时（>100 个），计算可能需要几秒
- 自动优化线程数时，建议设置合理的最大线程数（10-20）
- 超大规模任务建议使用专业的项目管理软件

#### 数据备份
- 定期导出 CSV 文件备份
- 使用版本控制工具（如 Git）管理任务文件
- 重要项目建议保存多个版本

#### 最佳实践
1. **任务粒度**: 单个任务耗时建议在 1 小时 -1 周之间
2. **依赖关系**: 尽量减少不必要的依赖
3. **并行化**: 识别可以并行的任务
4. **迭代优化**: 多次调整对比不同方案
5. **资源平衡**: 利用浮动时间调配资源

## 💡 核心概念

### 关键路径 (Critical Path)

**定义**: 决定项目最短总工期的任务序列。

**特点**:
- ✅ 路径上的任务总耗时 = 项目最短工期
- ✅ 任何任务延迟都会导致总工期延迟
- ❌ 没有浮动时间，必须按时完成
- 🔴 在图中用**红色**标识

**优化策略**:
1. 缩短关键任务的耗时
2. 将关键任务拆分为可并行的子任务
3. 增加资源投入
4. 优化依赖关系

### 非关键路径 (Non-Critical Path)

**定义**: 不直接决定总工期的任务序列。

**特点**:
- ✅ 拥有浮动时间（Slack Time）
- ✅ 可以适当延迟而不影响总工期
- ✅ 资源紧张时可调整优先级
- 🔵 在图中用**蓝色**标识

**优化策略**:
1. 利用浮动时间平衡资源
2. 在资源受限时延后执行
3. 作为备用优化空间

### 浮动时间 (Float/Slack)

**定义**: 一个任务可以延迟但不影响总工期的时间。

**计算公式**:
```
浮动时间 = 最晚开始时间 - 最早开始时间
```

**类型**:
- **总浮动**: 不影响项目完工日期的延迟时间
- **自由浮动**: 不影响后续任务最早开始时间的延迟时间

### 循环依赖 (Circular Dependency)

**定义**: 任务间形成环形依赖关系（逻辑错误）。

**示例**:
```
A → B → C → A  (错误！)
```

**后果**:
- ❌ 无法确定执行顺序
- ❌ 系统会报错并拒绝计算
- ❌ 需要手动修正

**解决方案**:
1. 检查依赖关系是否合理
2. 移除不必要的依赖
3. 重新设计任务流程

## 🎓 进阶技巧

### 优化策略

#### 1. 关键路径优化
- **压缩法**: 缩短关键任务耗时（加班、增派人手）
- **并行法**: 将串行关键任务改为并行
- **重叠法**: 部分重叠前后任务（快速跟进）

#### 2. 资源平衡
- **削峰填谷**: 利用非关键路径浮动时间
- **资源平滑**: 避免资源需求大起大落
- **优先级调整**: 关键路径优先分配资源

#### 3. 风险管理
- **缓冲设置**: 在关键路径末端设置缓冲时间
- **分散风险**: 避免过多任务在同一条路径上
- **应急预案**: 为高风险任务准备备选方案

### 高级分析方法

#### 蒙特卡洛模拟
对任务耗时进行概率分布假设，模拟多种可能结果。

#### 挣值管理
结合成本和进度，进行综合绩效评估。

#### 敏捷方法
将大任务拆分为小迭代，逐步交付价值。
